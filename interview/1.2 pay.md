**项目名称：** **企业级聚合支付与资金结算中台 (Tencent HR助手核心链路)**

**项目描述：** 针对业务线多条异构支付渠道（微信/支付宝等）的接入痛点，主导建设统一的聚合支付网关平台。屏蔽底层第三方 API 差异，为上游业务提供标准化的支付、查询、回调与退款能力，确保核心资金链路的高可用与零资损。

**核心职责与技术亮点：**

- **高可用异构路由架构（Architecture）：** 彻底重构早期面条式的 API 调用代码。采用**策略模式 (Strategy Pattern) 与工厂模式**，将不同支付渠道的 API 接入“插件化”；制定标准化的统一支付规约，使新支付渠道的接入效率提升 50% 以上，实现核心代码的强解耦。
    
- **零资损的最终一致性保障（Consistency）：** 针对第三方 API 调用超时、异步回调丢失等“逆风”场景，设计并落地**本地消息表 + MQ（消息队列）延迟重试机制**。通过主动轮询第三方渠道的支付状态，实现状态机的最终一致性补偿，确保极端网络抖动下的资金安全与零掉单。
    
- **网关级熔断与防雪崩隔离（Stability）：** 引入**舱壁模式 (Bulkhead Pattern)** 理念，针对不同的第三方支付 API 配置独立的**线程池隔离**。配合超时阻断机制，彻底杜绝因单一外部支付渠道（如某银行接口宕机）响应缓慢而拖垮整个平台网关的雪崩效应。
