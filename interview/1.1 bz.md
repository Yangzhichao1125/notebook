**项目名称：海外体育实时数据与互动娱乐平台 - 核心数据引擎**

**项目描述：** 针对日活 60 万+、核心赛况峰值 QPS 突破 5000 的高频海外互动场景，主导最底层的“赛事实时数据引擎”从 0 到 1 的架构演进。彻底解决原单体架构在毫秒级数据更新与海量多维检索下的“极端读写冲突”瓶颈，为全球用户提供高可用、低延迟的实时业务支撑。

**核心职责与技术亮点：**

- **异构存储与 CQRS 实时同步架构（Architecture）：** 主导设计 MySQL + Redis + Elasticsearch 的读写分离底座。设计并落地“冷热初始化预热 + 实时流处理”双链路：通过 Kafka 以比赛 ID 为 Key 进行 Hash 路由保证消息的**局部有序消费**；并在缓存端自研 **Redis Lua 脚本结合时间戳**的乐观锁机制实现高并发更新的幂等性，彻底根除网络抖动带来的乱序与脏数据问题。
    
- **极限调优与高可用兜底设计（Optimization）：** 针对高并发读写冲击实施全链路极限调优。缓存端采用长周期 TTL 配合 Redis 集群裸扛瞬时流量，基于 Trade-off 权衡**果断舍弃本地缓存以绝对规避脏数据风险**，保障核心接口 TP99 < 50ms；搜索端重构 ES **宽表模型（彻底消除多表 Join）**并引入“60天热数据窗口”，将核心索引**分片数由 3 降至 1**，根除跨节点聚合（Scatter-Gather）开销，并基于 1s 刷新机制实现对 Redis 的近实时降级兜底。
    
- **全球化架构与 AOP 零侵入解耦（Engineering）：** 针对海外多语言高并发查询痛点，摒弃低效的运行时关联字典表方案。在存储层贯彻**“空间换时间”**策略，将多语言核心字段预翻译并**扁平化冗余存储**至 ES 宽表；在应用层，基于 **Spring AOP** 自研 `@i18n` 动态渲染注解，通过拦截 HTTP Header 动态替换与组装响应体，实现了国际化展示逻辑与核心代码的**完全解耦与零侵入**，大幅降低系统迭代与维护成本。
